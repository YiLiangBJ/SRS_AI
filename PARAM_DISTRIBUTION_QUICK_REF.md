# 📊 参数类型分布 - 快速参考

## 🎯 一句话解释

```
weight: 80 个, 24,976 参数
        ↑          ↑
     张量数量   参数总数
     (count)    (sum)
```

**80 个** = 模型中有 80 个名为 `weight` 的参数张量  
**24,976 参数** = 这 80 个张量总共包含 24,976 个标量参数

## 💡 类比理解

| 概念 | 类比 | 说明 |
|------|------|------|
| **张量数量** | 你有几个钱包 | 80个weight张量 = 80个"钱包" |
| **参数总数** | 钱包里有多少钱 | 24,976个参数 = "总共24,976元" |
| **统计** | "80个钱包，总共24,976元" | "80个weight张量，总共24,976个参数" |

## 📐 具体示例

```python
# 模型中的参数张量
conv1.weight:  shape (8, 2, 3)   = 48 个参数    ← 第1个weight张量
bn1.weight:    shape (8,)        = 8 个参数     ← 第2个weight张量  
conv2.weight:  shape (8, 8, 3)   = 192 个参数   ← 第3个weight张量
...
conv40.weight: shape (...)       = ... 个参数   ← 第80个weight张量

# 统计结果
张量数量: 80 个 weight 张量
参数总数: 48 + 8 + 192 + ... = 24,976 个参数
```

## 📊 输出格式

```
【参数类型分布】
类型          张量数量      参数总数      占比
--------      --------      --------      ----
weight            80        24,976     97.7%   ← weight类型的统计
bias              40           578      2.3%   ← bias类型的统计
--------      --------      --------      ----
总计             120        25,554    100.0%   ← 所有参数

💡 说明：
   - 张量数量：该类型参数在模型中出现的次数
   - 参数总数：这些张量包含的标量参数总和
```

## 🔍 为什么要分开统计？

| 维度 | 张量数量 | 参数总数 |
|------|---------|---------|
| **含义** | 有多少个参数层 | 有多少个标量参数 |
| **反映** | 模型深度/结构 | 模型规模/容量 |
| **用途** | 结构分析 | 内存/计算量估算 |
| **示例** | 80个→约80层有weight | 24,976→约25K参数 |

## ✅ 快速检查

看到这个统计时，你应该知道：

1. **张量数量（80个）** 
   - ✓ 模型有约80个需要weight的层
   - ✓ 如果是ComplexUNet，实际可能是40个复数层（每个含2个weight：real+imag）

2. **参数总数（24,976个）**
   - ✓ 模型总共需要存储约25K个浮点数作为weight
   - ✓ 内存占用：24,976 × 4 bytes ≈ 97 KB（float32）

3. **占比（97.7%）**
   - ✓ weight参数占了绝大部分
   - ✓ 这是正常的，因为weight矩阵通常比bias向量大得多
